Submission & API

Option A (preferred): upload predictions JSON to a signed URL.

POST REPLACE_UPLOAD_ENDPOINT Authorization: Bearer REPLACE_API_KEY Content-Type: application/json { "run_id": "lab-YYYYMMDD-01", "predictions": [ {"problem_id":"SS-EN-00123","answer":"3.14 N","reasoning":"..."}, {"problem_id":"SS-RU-00456","answer":"E = mc^2","reasoning":"..."} ] }

Option B: request a batch /problems?split=test&limit=K, return predictions within 72h.

Rate limit: 60 req/min. Keys are per-team and revocable. No scraping; we can enable a separate crawlable sandbox on request.

Scoring

  • Numeric: tolerance window + units check + sig-fig sanity.
  • Symbolic: canonicalization with SymPy where viable; exact match otherwise.
  • Proof-style: rubric-based heuristic (claims/lemmas present, key step coverage) + spot human audit on a sample.
  • Partial credit: (+1) correct final; (+0.5) correct method w/ algebra slip; (+0.2) correct setup only.

Scoring spec · Baselines

What your lab receives

  • Overall accuracy and partial-credit score with confidence intervals.
  • Error taxonomy (units/dimensions, algebra, physical model, boundary conditions, misread, hallucination).
  • Topic × difficulty curves; RU vs EN contrast; numeric vs proof breakdown.
  • Optional latency summary (if provided).
  • Short public, aggregate report (lab-approved) + citation text.

We do not publish your prompts, weights, or per-item outputs; only aggregate metrics/breakdowns.

Timeline (4 weeks)

  3. Week 2–3: one iteration focusing on top error modes.

FAQ

Can we cite/announce participation?

Yes—after we agree on wording. Typical: “We evaluated our model on an open, education-only physics benchmark maintained by SavchenkoSolutions.”

Do you need our prompts or internals?

No. We only need final predictions (and optional latencies) for the test set.

Will our per-item mistakes be public?

No. Only anonymized, aggregate error breakdowns are public.

What about RU/EN balance?

You can choose EN-only, RU-only, or mixed; we’ll report language-specific metrics if desired.
